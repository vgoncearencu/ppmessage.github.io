PPMessage-Configurations

2016-05-05 |

前言

首次运行PPMessage之前，你需要生成PPMessage的配置文件。配置文件定义了PPMessage的管理员，nginx，数据库，客户端等设置选项。

准备工作

首先需要下载PPMessage源码。我们将源码放置在~/Documents目录下。

下载完成后，复制配置文件模板作为PPMessage的配置文件。

配置文件说明

配置文件位置为 ~/Documents/ppmessage/ppmessage/bootstrap/config.py，

配置文件里有如下设置项目：

超级管理员

此用户是超级管理员，可以管理PPMessage下的所有用户和客服团队。各项设置介绍如下：

user_language: 管理员的语言设置，决定系统创建PPCom匿名用户时，用何种语言生成用户名，以及PPCom的默认欢迎信息语言。可选zh_cn(简体中文), zh_tw(繁体中文), en_us(英文)。
user_firstname: 管理员的名
user_lastname: 管理员的姓
user_fullname: 管理员的全名
user_email: 管理员的登录邮箱
user_password: 管理员的登录密码

超级管理员客服团队

此团队是超级管理员的客服团队，各项设置解释如下：
app_name： 客服团队的名称
company_name： 公司或组织名称

mysql 设置

此处设置PPMessage登录mysql时所需要的参数，各项设置解释如下：
db_host: mysql server地址，默认连接本地数据库，值为127.0.0.1，无需修改
db_user: mysql 登录用户名，默认为root，无需修改
db_pass: mysql 登录密码，需要将其设置为你的mysql登录密码（安装mysql时会提示设置密码），如果使用Docker搭建PPMessage开发环境，则密码必须为test
db_name: ppmessage数据库名，默认为ppmessage，无需修改

服务器设置

此处设置PPMessage服务器IP，文件存储路径。各项设置解释如下：
name: 服务器IP，例如191.168.0.110，PPKefu需要读取此设置，决定要连接哪个服务器
